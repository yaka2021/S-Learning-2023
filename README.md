# ブランチの扱い方
各種ブランチの扱い方です。

## masterブランチ
**masterブランチは本番環境です。変更された内容は自動的にロリポップのディレクトリに反映されます。**

masterブランチへの直接のpushは絶対に避けてください。編集内容を反映したい場合はdevelopブランチをmargeしてください。

※margeした際、developブランチは**削除しない**でください

## developブランチ

**developブランチは作業環境です。メンバーが編集した内容は一度このブランチに集約され、作業状況を共有・整理した後、masterに反映することを目的としています。**

developブランチへの直接のpushは避けてください。編集内容を反映したい場合は、developブランチをローカルにクローンした後、developから派生する新しいブランチをローカルで作成し、編集後、developにmargeしてください。

※margeした際、個人で作成したブランチは**削除して**ください
